Git commit message 的編輯器裡，`#` 預設是註解符號，存檔後這一行不會被納入 commit message。

大多數時候沒問題，但如果你的 commit message 需要用 `#` 開頭——例如引用 issue 編號 `#123 Fix login bug`——Git 就會把整行吃掉，留下一個空的 commit message。

---

## 解法

把 Git 的 commit 註解符號改成其他字元，例如 `;`：

```shell
git config --global core.commentchar ';'
```

改完之後，commit 編輯器裡改以 `;` 開頭的行才會被視為註解，`#` 就可以自由使用了。

---

## 只改單一 repo

如果不想改全域設定，拿掉 `--global` 就只會影響當前 repo：

```shell
git config core.commentchar ';'
```
